Remarks

You can either use RomanNumber to store a roman number by creating instances of it or use its static methods to convert from/to roman numbers.

See https://en.wikipedia.org/wiki/Roman_numerals for details about roman numbers.

Only the uppercase letters I (1), V (5), X (10), L (50), C (100), D (500), M (1000) are supported, lowercase letters of those characters are considered invalid (as is any other character).

When converting from roman to decimal, additive and subtractive forms are supported. When converting from decimal to roman, the subtractive form is used for certain special cases (see below).

The following special cases represent numbers in subtractive form (only these are supported): IV (4), IX (9), XL (40), XC (90), CD (400), CM (900).

Both positive and negative numbers are supported as well as zero. For positive values, DecimalToRoman(Int32) and RomanValue will never add the plus sign as a prefix but parsed values (Parse(String), TryParse(String, RomanNumber), DecimalToRoman(Int32)) can have an optional plus sign prefix. For negative values, both roman and decimal representations use a minus sign prefix. An empty string is used to express zero.

 Important
Apostrophus and Vinculum are not supported for larger numbers. Therefore, the value 1'000'000'000 would result in one million M s. It is advised that the numbers are clamped to a reasonable range which is suitable for you.

Copy
 1// convert decimal to roman
 2var roman = RomanNumber.DecimalToRoman(123); // Result: "CXXIII"
 3
 4// convert roman to decimal
 5var dec = RomanNumber.RomanToDecimal("CLIV"); // Result: 154
 6
 7// create two roman numbers
 8RomanNumber num1 = 123;
 9RomanNumber num2 = "CLIV";
10
11// do some math with roman numbers
12RomanNumber num3 = num1 + num2;
13
14// get explicit values
15var romanValue = num3.RomanValue; // "CCLXXVII"
16var decimalValue = num3.DecimalValue; // 277
