Remarks

The EqualityComparisonT created with this constructor uses the objects (of type T) own GetHashCode method for calculating hash values.

Therefore, the EqualityComparisonT created with this constructor should not be used in scenarios where hash values are used, e.g. as the equality comparer of a HashSetT or DictionaryTKey, TValue, if the hash value calculation of the objects own GetHashCode method does not correspond to the equality comparison performed by comparer. If those do not correspond but it is still used for hash values, performance might be severly impacted or, depending on the usage of the hash values, other strange side-effects might occur.
