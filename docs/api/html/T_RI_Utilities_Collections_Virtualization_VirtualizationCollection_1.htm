Remarks

VirtualizationCollectionT uses an IItemsProviderT or INotifyItemsProviderT to load items on-demand. That means that items are only loaded when they are actually requested through VirtualizationCollectionT (e.g. by using the collections indexer property).

Items are loaded in pages which size can be specified when constructing VirtualizationCollectionT. The loaded pages will then stay in the cache for a specified amount of time.

 Note
If INotifyItemsProviderT is used, ItemsChanged will clear the entire cache.
Thread Safety
Static members of this type are not safe for multi-threaded operations. Instance members of this type are not safe for multi-threaded operations.
