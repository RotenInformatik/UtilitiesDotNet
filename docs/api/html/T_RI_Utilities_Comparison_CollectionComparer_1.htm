Remarks
 Note
Two collections are considered equal if they contain the same number of elements and each element in one collection has an equal element in the other collection. The collections themselves are not compared for equality, only their elements.
 Important
This implementation of IEqualityComparerT is only intended for equality comparison of elements of collections. It should not be used in scenarios where hash values of the collections themselves are used, e.g. used as a hash provider for the collections when storing the collections in another collection (e.g. storing collections in a HashSetT or DictionaryTKey, TValue).
Examples
C#
Copy
 1// create some collections of strings we want to compare
 2var upperCaseAscending  = new List<string>    { "ANDREW", "BOB", "CHARLES" };
 3var lowerCaseDescending = new List<string>    { "charles", "bob", "andrew" };
 4var mixedCase           = new HashSet<string> { "Andrew", "Charles", "Bob" };
 5
 6// create collection comparer with case-insensitive string comparison
 7var comparer = new CollectionComparer<string>(StringComparer.OrdinalIgnoreCase);
 8
 9comparer.Equals(upperCaseAscending, lowerCaseDescending); // returns "false"
10comparer.Equals(upperCaseAscending, mixedCase);           // undefined, HashSet<T> does not guarantee the order its elements are enumerated
11comparer.Equals(lowerCaseDescending, mixedCase);          // undefined, HashSet<T> does not guarantee the order its elements are enumerated
12
13// create collection comparer with case-insensitive string comparison and ignore order
14comparer = new CollectionComparer<string>(CollectionComparerFlags.IgnoreOrder, StringComparer.OrdinalIgnoreCase);
15
16comparer.Equals(upperCaseAscending, lowerCaseDescending); // returns "true"
17comparer.Equals(upperCaseAscending, mixedCase);           // returns "true"
18comparer.Equals(lowerCaseDescending, mixedCase);          // returns "true"
