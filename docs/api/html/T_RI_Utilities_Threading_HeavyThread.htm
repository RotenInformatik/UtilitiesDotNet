Remarks

HeavyThread is usually used for long-running and/or dedicated threads where a higher control and encapsulation of the threads execution is required, e.g. compared to ThreadPool.

See Start and Stop for a description of the thread execution sequence.

 Important
Some virtual methods are called from within locks to SyncRoot. Be careful in inheriting classes when calling outside code from those methods (e.g. through events, callbacks, or other virtual methods) to not produce deadlocks!
Thread Safety
Static members of this type are safe for multi-threaded operations. Instance members of this type are safe for multi-threaded operations.
