Remarks

A request/response collection is a thread-safe, asynchronous, bidirectional producer/consumer collection.

On one side is one or more producers which can, from any thread, add requests to the collection. After a request is added, a response is awaited.

On the other side is one or more consumers which can, from any thread, take requests from the collection. A consumer then processes a request and issues the response, causing the awaiting request to continue with that response.

Therefore, requests and responses can be two different types (TRequest and TResponse) to allow true bidirectional data flow from the producer to the consumer and back.

A consumer can only process one request at a time. And one request is processed by exactly one consumer.

If there are consumers waiting for requests, a new request is issued to the consumer already waiting the longest. If there are no consumers waiting for requests, a new request will be stored and issued to the next consumer which wants to take requests for processing.

Requests are presented to consumers using types deriving from RequestResponseItemTRequest, TResponse (TItem). RequestResponseItemTRequest, TResponse wraps the request (available through the Request property) and provides methods to finish the request processing and issue a response (such as Respond(TResponse)).

Thread Safety
Static members of this type are safe for multi-threaded operations. Instance members of this type are safe for multi-threaded operations.
